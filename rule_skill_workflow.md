Antigravity(Claude Code의 개발 코드네임) 또는 현재 정식 명칭인 **Claude Code**에서 **Rule, Workflow, Skill**은 에이전트의 행동 방식을 결정하는 핵심 레이어들입니다.

간단히 비유하자면, **Rule**은 도덕 책(지침), **Skill**은 도구 상자(능력), **Workflow**는 작업 설명서(절차)라고 이해하시면 됩니다.

---

## 1. Rules (규칙/지침)
**Rule**은 에이전트가 작업을 수행할 때 반드시 지켜야 하는 **행동 강령**이자 **제약 조건**입니다. 

* **목적:** 코드 스타일 유지, 특정 라이브러리 사용 금지, 보안 가이드라인 준수 등.
* **특징:** "무엇을 할 수 있는가"보다는 **"어떻게 행동해야 하는가"**에 집중합니다.
* **예시:**
    * "모든 함수에는 JSDoc 주석을 달아야 함."
    * "테스트 코드는 반드시 `vitest`를 사용해야 함."
    * "커밋 메시지는 Conventional Commits 형식을 따를 것."

## 2. Skills (기술/도구)
**Skill**은 에이전트가 운영체제나 코드베이스와 상호작용하기 위해 사용하는 **실행 가능한 도구(Tools)**입니다.

* **목적:** 실제로 무언가를 실행하거나 정보를 가져오는 기능을 제공.
* **특징:** 에이전트가 직접 손발을 움직이는 기능입니다. 주로 MCP(Model Context Protocol)를 통해 확장되기도 합니다.
* **예시:**
    * `ls`: 디렉토리 구조 확인.
    * `grep`: 특정 패턴 검색.
    * `edit_file`: 파일 수정.
    * `run_terminal_command`: 빌드 명령어나 테스트 실행.

## 3. Workflows (워크플로우/절차)
**Workflow**는 특정 목표를 달성하기 위해 **Rule을 준수하며 여러 Skill을 조합하는 일련의 과정**입니다.

* **목적:** 복잡하고 반복적인 작업을 자동화된 시나리오로 정의.
* **특징:** "A를 조사한 뒤, 결과가 B라면 C를 실행하고 리포트를 작성하라"와 같은 논리적 흐름입니다.
* **예시:**
    * **버그 수정 워크플로우:** 오류 로그 분석 → 관련 파일 탐색 → 테스트 코드 작성 → 코드 수정 → 테스트 통과 확인.
    * **리팩토링 워크플로우:** 중복 코드 검색 → 추상화 모델 설계 → 일괄 교체 → 영향도 평가.

---

### 요약 비교표

| 구분 | 개념 | 중심 요소 | 비유 |
| :--- | :--- | :--- | :--- |
| **Rule** | 가이드라인 | 정책 및 스타일 (Policy) | 헌법/법률 |
| **Skill** | 실행 도구 | 기능 및 API (Function) | 장비/도구 |
| **Workflow** | 작업 프로세스 | 논리 및 순서 (Process) | 프로젝트 계획서 |

이 세 가지가 결합되어, Claude Code는 단순히 텍스트를 생성하는 모델을 넘어 **"규칙을 지키며(Rule), 도구를 사용하여(Skill), 복잡한 문제를 해결(Workflow)"**하는 능동적인 개발 에이전트로 동작하게 됩니다.
