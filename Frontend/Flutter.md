# Flutter

- Google 제작. 하나의 코드로 iOS와 Android를 동시에 개발. UI가 매우 미려함.

**Flutter(플러터)**는 구글이 개발한 오픈소스 UI 소프트웨어 개발 키트(SDK)로, 하나의 코드베이스로 **모바일(iOS, Android), 웹, 데스크톱, 임베디드 기기**까지 아우르는 고성능 애플리케이션을 만들 수 있게 해줍니다.

2026년 현재, Flutter는 단순한 크로스 플랫폼 프레임워크를 넘어 더욱 성숙하고 강력한 생태계를 구축하고 있습니다. 주요 특징과 장단점을 정리해 드릴게요.

---

## 1. 주요 특징

- **하나의 소스 코드:** 한 번 작성한 코드로 여러 플랫폼에 동시에 배포할 수 있어 개발 비용과 시간이 획기적으로 절감됩니다.
- **빠른 개발 (Hot Reload):** 코드를 수정하면 앱을 재시작하지 않아도 즉시 화면에 반영됩니다. (2026년 현재는 더욱 고도화되어 상태 유지 능력이 매우 뛰어납니다.)
- **고성능 렌더링 엔진 (Impeller):** 기존 Skia 엔진을 대체하는 **Impeller** 엔진이 안드로이드와 iOS에 완전히 정착하여, 프레임 드랍(Jank) 없는 부드러운 애니메이션을 제공합니다.
- **Widget 중심 설계:** 모든 UI가 '위젯(Widget)'으로 구성되어 있어, 복잡한 UI도 레고 블록을 쌓듯 쉽고 정교하게 커스텀할 수 있습니다.
- **Dart 언어 사용:** 정적 타이핑과 가비지 컬렉션이 강력한 Dart 언어를 사용합니다. 최근에는 **Wasm(WebAssembly)** 지원이 기본이 되면서 웹 성능이 네이티브 수준에 도달했습니다.

## 2. 2026년의 새로운 트렌드

- **GenUI (생성형 UI):** AI 모델이 사용자 의도에 따라 실시간으로 UI를 생성하고 최적화하는 '에이전틱(Agentic) 앱' 아키텍처가 도입되고 있습니다.

- **고급 3D 및 AR 지원:** 단순한 2D UI를 넘어 가구 배치, 가상 피팅 등 실무 환경에서 사용할 수 있는 3D/AR 기능이 표준화되었습니다.
- **Wasm 기반 웹 성능:** 웹 환경에서 자바스크립트의 한계를 넘어선 네이티브급 실행 속도를 보장합니다.

## 3. 장점과 단점

| 구분 | 내용 |
| :--- | :--- |
| **장점** | 생산성 극대화, 플랫폼 간 UI 일관성 완벽(Pixel Perfect), 강력한 커뮤니티 및 라이브러리 |
| **단점** | 네이티브 앱 대비 비교적 큰 앱 용량, OS 고유의 최신 기능(예: 새로운 iOS 전용 위젯) 적용 시차 발생 |

---

## 4. Flutter를 선택해야 할 때

1. **빠른 시장 진입(MVP)**이 필요한 스타트업
2. **iOS와 Android에서 완전히 동일한 브랜드 디자인**을 유지해야 할 때
3. **적은 인원**으로 다양한 플랫폼(모바일, 웹, 데스크톱)을 운영해야 할 때

> **💡 한 줄 요약:** Flutter는 "하나만 배워서 모든 곳에 수준 높은 앱을 만들 수 있게 해주는" 현대 개발자들의 가장 강력한 무기 중 하나입니다.
